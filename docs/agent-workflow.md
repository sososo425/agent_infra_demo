# Agent 工作流 SOP

> **read_when**: 开始一个新任务、讨论 Agent 工作流程、需要理解 Plan-Execute-Verify 模式时。

## 概述

本文档定义了 AI Agent 在本项目中执行任务的标准作业流程（SOP）。
核心思想：**Plan → Execute → Verify**，对应 Idea.md 中的角色分层模型（Orchestrator → Planner → Executor）。

## 工作流程

### Phase 1: Plan（规划）

在动手写代码前，必须完成以下步骤：

1. **理解需求**: 明确任务目标、范围和验收标准
2. **查阅文档**: 根据 `AGENTS.md` 的索引，阅读相关的知识库文档
3. **制定方案**: 列出实现步骤、涉及的文件、可能的风险
4. **确认方案**: 将方案呈现给人类确认（除非是非常简单的任务）

**产出物**: 一段清晰的实现计划（自然语言）

### Phase 2: Execute（执行）

按照确认的方案逐步实施：

1. **小步迭代**: 每次修改聚焦一个关注点，控制在 3-5 个文件内
2. **先写测试**: 对于新功能，先写测试用例定义"完成标准"
3. **编写代码**: 遵循 `.cursor/rules/` 中的编码规范
4. **即时验证**: 每完成一个小步骤就运行测试

**产出物**: 可工作的代码 + 测试

### Phase 3: Verify（验证）

实施完成后的检查清单：

- [ ] 所有新增代码有测试覆盖
- [ ] `ruff check` 通过（无 lint 错误）
- [ ] `ruff format --check` 通过（格式正确）
- [ ] `mypy` 类型检查通过
- [ ] 无硬编码的敏感信息
- [ ] 无 TODO/FIXME 残留
- [ ] 相关文档已更新（如有必要）

**验证命令**:

```bash
make check    # 一键运行所有检查（待配置 Makefile）
```

## 任务规模指引

| 任务规模 | 文件影响 | 是否需要 Plan 确认 | 示例 |
| :--- | :--- | :--- | :--- |
| 小 (S) | 1-2 个文件 | 不需要 | 修复 typo、调整配置 |
| 中 (M) | 3-5 个文件 | 建议确认 | 新增 API 接口、重构模块 |
| 大 (L) | 5+ 个文件 | 必须确认 | 架构变更、新增子系统 |

## 中断与恢复

当任务因为对话过长需要中断时：

1. 在对话中总结当前进度和下一步计划
2. 新对话开始时，读取 `AGENTS.md` 获取项目上下文
3. 如有进度文件（`docs/progress.md`），优先阅读
